# Week 10 — Deployment and Submission

## Lesson 28: Final Polish

**Goal:** Ensure site is error-free.

**Activity:**
- Full-site render.
- Verify links, images, formatting consistency, and chart labels.
- Complete final self-check against rubric.

**Teacher checks:**
- No critical rendering or navigation errors.

---

## Lesson 29: Publishing to the Web

**Goal:** Make the site live.

**Activity:**
- Publish static output via your local workflow:
  - render locally
  - push `docs/` to GitHub
  - host with GitHub Pages (`main` → `/docs`)
- (Optional alternative) Quarto Pub demo if needed.

**Teacher checks:**
- Live URL accessible.

---

## Lesson 30: Compilation and Hand-in

**Goal:** Finalize official submission package.

**Activity:**
- Compile:
  - live URL
  - source files (`.qmd`, `_quarto.yml`, assets)
  - wireframes
  - testing table/development log
  - evaluation + A/M/E evidence statement
- Submit as required by school process.

**Teacher checks:**
- Submission set is complete and attributable.
